Chapter Summary

Chapter 1 — IPv4 Summary

Purpose & structure

IPv4 enables unicast communication (one device to one) and is hierarchical—unlike flat MAC addresses, it reveals the network and the unique host. With a subnet mask, it identifies both.

Format

  • 32-bit address in 4 octets (e.g. 192.168.1.1)
  • Binary: bits double left to right — 1, 2, 4, 8, 16, 32, 64, 128 per octet
  • 11111111 = 255

Network mask

  • 1 = network bit, 0 = host bit
  • Same info three ways: binary, decimal (255.255.255.0), slash (/24)

Critical rule

  • Host all 0s → network address
  • Host any mix → valid host
  • Host all 1s → broadcast

192.168.1.1/24: network 192.168.1.0, hosts .1–.254, broadcast 192.168.1.255. 254 usable hosts (28 − 2).

Classes A–E

  • A: 0xxxxxxx, 0–127, 10.0.0.0/8 private
  • B: 10xxxxxx, 128–191, 172.16.0.0/12 private
  • C: 110xxxxx, 192–223, 192.168.0.0/16 private
  • D: 224–239 (multicast), E: 240–255 (reserved)
  • Hosts: A = 224−2, B = 216−2, C = 254

RFC 1918 & NAT

Private addresses are internal only; need NAT for public access.

Local vs remote

Same network = local (direct). Different networks = remote (requires router or L3 switch).

What are IPv4 addresses used for?

  • Created to allow one specific digital device to communicate directly with another specific digital device
    • This is called unicast communication
  • Needs to allow for one device to access multiple devices

An IPv4 address is known as a hierarchical address because it can be divided into multiple smaller domains.

An IPv4 address in conjunction with a subnet gives you two pieces of information:

  • The network in which the device is located
  • The unique identifier of the device
Note: Different from a MAC address—a MAC address is a flat address. It does not give information about the network it's connected to.

What does an IPv4 address look like?

  • The computer sees a 32-bit number
  • For us they are broken up into 4 groups of 8 bits called bytes or octets, separated by periods
  • For example:
    • 11000000.10101000.00000001.00000001
    • This would result in 192.168.1.1

More about binary (the type of language this is)

  • You start with 00000000
  • The first bit stands for 1, second for 2, third for 4, fourth for 8, fifth for 16, sixth for 32, seventh for 64, eighth for 128
  • This is assuming an 8-bit number; the pattern continues (keep doubling)
Examples: 00011111 = 31, 11111111 = 255. Practice: 11010011 = 227

IPv4 needs some way to differentiate the network portion from the host portion. All IPv4 addresses have a network mask (subnet mask).

A network mask is 32 bits that define what portion represents the network and which portion is the host.

Rules for network masks

  • A 1 in the mask defines a network bit
  • A 0 in the mask defines a host bit

3 ways to denote a subnet mask

  1. Binary: 11111111.11111111.11111111.00000000
  2. Decimal: 255.255.255.0
  3. Slash notation: /24

All tell us the first 24 bits are 1 and the last 8 are 0. The first 24 bits are the network portion; the last 8 define the host.

Example: 192.168.1.1 + 255.255.255.0 → 192.168.1.1/24
Key rule: In an IP address, host portion all 0s = network itself; combination = specific device; all 1s = broadcast address.
192.168.1.1/24: Network = 192.168.1.0, Hosts = 192.168.1.1–192.168.1.254, Broadcast = 192.168.1.255

Network vs. Node (Host) Bits

Within classes A–C: network (N) vs node/host (H) octets. Class A = /8, B = /16, C = /24.

  • Class A: 224 − 2 = 16,777,214 hosts/network
  • Class B: 216 − 2 = 65,534 hosts/network
  • Class C: 28 − 2 = 254 hosts/network

Conclusions: 126 Class A networks • 16,384 Class B • 2,097,152 Class C

Example: 192.168.1.1/24 is a Class C network.

RFC (private) 1918 Addresses

  • Created to help slow the depletion of IPv4 addresses
  • Addresses that are part of RFC are to be used for internal networks only. They are allowed within the network but not allowed to the public.
  • Must go through a Network Address Translation (NAT) to be used for public use.

Local vs. Remote Addresses

  • Addresses within the same network and can communicate are considered Local to each other
  • Addresses in two different networks that can communicate are considered remote to each other
  • Local devices can only communicate directly with each other
  • Remote devices require a Layer 3 device (such as a router or layer 3 switch)

Lessons Learned

  • 192.168.1.1/24 tells us:
    • The network portion is 192.168.1
      • This is shown as 192.168.1.0/24
    • The host portion of this address is .1
    • There are 254 valid hosts on this network
      • This is the first valid host
    • The broadcast address is 192.168.1.255/24
    • This is a Class C network

Chapter 2 — Subnetting Summary

Goal

Split a Class C network (e.g. 192.168.100.0/24) into smaller subnets. Borrow only from host bits (4th octet); network bits stay fixed.

Step 1 — Bits to borrow

  • 2N ≥ subnets needed
  • Nine subnets → 24 = 16 ≥ 9, so borrow 4 bits
  • 8 H bits become SSSSHHHH (4 subnet, 4 host)
  • Each subnet: 24−2 = 14 hosts

Step 2 — Binary pattern

  • Subnet number = all 0s in host portion; broadcast = all 1s
  • First subnet: 0–15; second: 16–31; increment by 16

Step 3 — IP plan table

  • Count by 16: Subnet 1 = .0, hosts .1–.14, broadcast .15
  • Subnet 2 = .16, hosts .17–.30, broadcast .31
  • Continue: .32, .48, .64 … up to .240

New subnet mask

Original /24 → 255.255.255.0. Borrow 4 bits → /28 = 255.255.255.240.

Quick checks

Network = even; first host = odd; last host = even; broadcast = odd. 16 subnets total.

How to Subnet IPv4 Addresses

Subnetting a Class C network

Say you have a network address of 192.168.100.0/24 and you need nine subnets. You can only use H bits, not N bits. So you can ignore the 192.168.100 part.

Remember for Class C networks that it's like such:
